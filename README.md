# CrawlTrip is an AI-Aware Reverse Proxy

CrawlTrip is a Go-based reverse proxy designed to identify, classify, and monitor requests generated by AI Crawlers and AI Assistants while transparently forwarding traffic to an upstream application.

The project demonstrates modern backend and infrastructure concepts including reverse proxying, asynchronous messaging, distributed system decoupling.

The project represents the capstone of the Boot.dev Golang course, showcasing the application of learned concepts in a real-world inspired scenario. I think this may have no utility beyond being a learning exercise, but it was a fun and educational project to build!

## Motivation

Audit and analitycs tools often used to monitor web traffic can be low in fidelity and may not accurately capture the behavior of AI crawlers and assistants. CrawlTrip aims to provide a more accurate and reliable solution for identifying and classifying AI-generated requests, enabling better monitoring and analysis of web traffic.

## Features

* HTTP reverse proxy based on `net/http/httputil.ReverseProxy`
* AI bot detection with monperrus/crawler-user-agents pkg
* Reliable asynchronous event processing using NATS JetStream with durable consumers and message persistence.
* MongoDB request storage with a simple schema for request metadata
* Dead-letter queue for failed message processing
* Graceful shutdown and service lifecycle management

## Quick Start

### Prerequisites
* Go 1.20+
* Docker

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/LoSquadrato/crawltrip.git
   cd crawltrip
   ```
2. Use shell scripts to build and run the services:
   ```bash
   ./nats.sh start
   ./mongo.sh start
   ```
3. Check the services are running:
   ```bash
   ./nats.sh status
   ./mongo.sh status
   ```
4. Create a .env file with the following content:
   ```env
   ADMIN_SECRET=your_admin_secret
   ```
5. Build and run the servers and worker services:
   ```bash
   go run ./cmd/proxy
   go run ./cmd/worker
   go run ./cmd/demo-server
   ```
6. Check the services are running:
   ```bash
   curl http://localhost:8080/proxy/ping -H "X-Admin-Secret: your_admin_secret"
   ```

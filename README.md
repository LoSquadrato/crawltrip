# CrawlTrip is an AI-Aware Reverse Proxy

CrawlTrip is a Go-based reverse proxy designed to identify, classify, and monitor requests generated by AI Crawlers and AI Assistants while transparently forwarding traffic to an upstream application.

The project demonstrates modern backend and infrastructure concepts including reverse proxying, asynchronous messaging, distributed system decoupling, external API integration, and automated bot policy generation.

The project represents the capstone of the Boot.dev Golang course, showcasing the application of learned concepts in a real-world inspired scenario. I think this may have no utility beyond being a learning exercise, but it was a fun and educational project to build!

## Features

* HTTP reverse proxy based on `net/http/httputil.ReverseProxy`
* AI bot detection through User-Agent classification
* Integration with Cloudflare Radar bot intelligence
* Reliable asynchronous event processing using NATS JetStream with durable consumers and message persistence.
* MongoDB request storage
* Automated generation of:
  * `robots.txt`
  * `llms.txt`
  * `bots.txt`
* Request metadata collection and analytics via a simple API managed by Gin
* Graceful shutdown and service lifecycle management

## Architecture

```text
Client
  |
  v
Reverse Proxy
  |
  +--> Bot Classifier
  |        |
  |        +--> AI Crawler / Assistant
  |
  +--> NATS Event Bus
              |
              v
        Storage Worker
              |
              v
           MongoDB

  |
  v
Upstream Server
```

The proxy remains lightweight by publishing request events to NATS and delegating persistence to a dedicated worker service.

## Components

### Proxy Service

Responsible for:

* Accepting incoming HTTP requests
* Classifying clients
* Publishing AI-related events to NATS
* Forwarding requests to the upstream target

### Worker Service

Responsible for:

* Consuming NATS events
* Persisting request metadata
* Performing background processing

### Bot Filter

Responsible for:

* Fetching bot intelligence from Cloudflare Radar
* Maintaining a local bot cache
* Categorizing requests

Supported categories:

* Human
* Search Engine Bot
* AI Crawler
* AI Assistant
* Unknown Bot
